Blog Factory

AI Agent 驱动 + 人类决策把关的半自动内容生产流水线
为 nexu 品牌打造深度 SEO 内容

01设计思想

不是全自动内容农场，而是一个有明确分工的人机协作系统

🤖 机器负责

  • 数据抓取（4 个免费数据源）
  • 相关性打分（LLM 评估）
  • 长文写作（商业叙事结构）
  • 封面生成（Figma MCP）
  • 关键词去重（自动历史记录）

👤 人类负责

  • 选题判断（Editorial Judgment）
  • 质量把关（Quality Gate）
  • 品牌调性控制
  • 反幻觉确认
  • 配置调优（种子词 / 阈值）
核心理念："写什么"比"怎么写"更重要。选题留给人，是刻意的设计——这是 editorial judgment，不适合全交给 AI。

02技术栈

轻量级 Python 流水线 + LiteLLM 统一模型接口

层级技术选型说明
语言 Python 3 脚本式流水线，无重型框架依赖
数据源 Google Trends / HN / GitHub / Reddit 全部免费，无需 API Key
LLM OpenAI SDK → LiteLLM Proxy 统一端点，底层可切换任意模型
打分模型 Claude 3.5 Haiku 便宜快速，适合批量评估
写作模型 Claude Sonnet 4 高质量长文生成
配置 config.yaml 单文件集中管理所有参数
编排 orchestrator.py CLI Cursor Agent 代执行命令
封面 Figma MCP 模板克隆 + 文字填充 + 导出 PNG
去重 data/history.json 已写关键词自动记录，避免重复

03四步流水线

Discover → Score → Select → Produce — 数据从左到右流动

Step 1

Discover

从 Google Trends、Hacker News、GitHub Trending、Reddit 并行抓取热点话题，自动去重已写关键词

Step 2

Score

LLM 对每个话题做相关性打分（0-100），筛出 ≥ 70 分的 Top 20 候选，输出评审表格

AI 执行
→
关键步骤

Select

人审阅候选表格，在 Cursor 对话中告诉 AI "选 1, 3, 5"，AI 写入 selected.json

人机协作
→
Step 3

Produce

SKILL.md 作为 system prompt，LLM 按商业叙事结构生成中英双语文章 + Figma 封面

AI 执行

Step 1: Discover — 热点发现

4 个免费数据源

  • Google Trends — 种子词: AI Agent, MCP protocol, AI workflow…
  • Hacker News — Algolia API 搜索相关帖子 Top 30
  • GitHub Trending — 每日趋势仓库抓取
  • Reddit — LocalLLaMA, LangChain, MachineLearning 等子版块

关键逻辑

  • 跳过 history.json 中已写过的关键词
  • 合并去重后按 heat_score 排序
  • 输出 output/topics/{date}-topics.json

Step 2: Score — 相关性打分

用 Claude 3.5 Haiku（便宜快速）对每个话题评估与 nexu 核心领域的关联度。LLM 同时输出 domain_match、suggested_angle、suggested_title。

参数值说明
relevance_threshold70低于此分数的话题被过滤
top_n_candidates20最终呈现给人审阅的候选数

输出两份文件：candidates.json（结构化数据）+ candidates.md（人类可读表格），以及一个可视化的 select.html 暗色主题审阅页面。

Select — 人工选题

唯一必须人工参与的步骤。有两种执行方式：

方式 A — CLI 交互：运行 orchestrator.py select，在终端查看列表输入编号
方式 B — Cursor 对话（推荐）：AI 展示表格 → 你说"选 1, 3, 5, 7" → AI 直接写 selected.json

Step 3: Produce — 文章生成

核心设计：把整个 SKILL.md（600 行）作为 system prompt 灌入 LLM。用 Claude Sonnet 4 按以下规范生成完整文章：

写作结构

  • 四段式商业叙事：痛点 → 介入点 → 流程重构 → ROI
  • 每篇含 Mermaid 业务流程图
  • 结尾必须有 "Start Now 3 步骤" 或 "Decision Checklist"
  • 中英文双版本独立写作（非翻译）

质量控制

  • 反 SEO 废话（禁用空洞开场白）
  • 每段 ≤ 4 句，每句推进论证
  • 所有建议一个人就能执行
  • 工具 / 框架必须给具体名称 + 链接
  • nexu 品牌自然融入，非硬广

封面生成 — Figma MCP

从文章中提取封面标题（英文 5-9 词 / 中文 9-15 字），通过 Figma MCP 在模板上克隆 + 填文字 + 导出 PNG。

06设计亮点

5 个关键设计决策及其背后的考量

SKILL.md = System Prompt

  • 写作规范不是文档，而是直接喂给 LLM 的指令
  • 改 SKILL = 改写作风格，零代码变更
  • 包含品牌指南、写作准则、禁用词表、Quality Gate

LiteLLM 统一接口

  • 所有 LLM 调用走同一个 OpenAI 兼容端点
  • 底层随时切换 Claude / GPT / 开源模型
  • 无需改代码，只改 config.yaml

分级模型策略

  • 打分用 Haiku（便宜 + 快速）
  • 写作用 Sonnet（质量 + 深度）
  • 成本和质量两手抓

自动去重机制

  • history.json 记录所有已写关键词
  • Discover 阶段自动跳过历史话题
  • 内容覆盖面随时间自然扩大

人机协作设计

  • 选题刻意留给人类（Editorial Judgment）
  • 在 Cursor 对话中完成，无需记命令
  • AI 负责执行，人负责决策

Cursor 原生工作流

  • 整个流水线在 Cursor IDE 中驱动
  • 用户说自然语言，AI Agent 跑脚本
  • 无需记忆任何 CLI 命令
